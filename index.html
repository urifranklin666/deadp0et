Operational notes

This client now reflects the backend contract more closely than the original in-browser mock.

Local-only secrets

Device private keys are generated in the browser and cached locally so this device can decrypt later.

Verifier flow

The current prototype sends a SHA-256-derived verifier, not a production-grade PAKE or Argon2id verifier.

Device-scoped inbox

The inbox request is bound to the authenticated session’s device so envelope delivery stays per-device.

Current gap

If a browser does not already have the local private keys for an account, it can authenticate but cannot decrypt old messages.
